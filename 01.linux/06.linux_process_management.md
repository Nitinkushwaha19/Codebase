# Process Management Commands

## 1. `top` – View Running Processes
Displays a dynamically updating list of running processes.

**Example:**
```bash
$ top
````

---

## 2. `ps` – View Processes

Shows a snapshot of current running processes.

**Example:**

```bash
$ ps aux
```

---

## 3. `kill` – Terminate a Process by PID

Stops or terminates a process using its process ID (PID).

**Examples:**

```bash
$ kill 156        # Graceful termination
$ kill -9 156     # Force termination
```

---

## 4. `killall` – Terminate Processes by Name

Stops all processes with a given name.

**Examples:**

```bash
$ killall top           # Graceful termination
$ killall -9 top        # Force termination
$ killall -KILL top     # Alternative force termination
```

# CPU Monitoring Commands

## 1. `lscpu` – CPU Architecture Info
Displays information about CPU architecture: cores, threads, sockets, and NUMA nodes.

**Example:**
```bash
$ lscpu
Architecture:        x86_64
CPU(s):              40
Thread(s) per core:  1
Core(s) per socket:  10
Socket(s):           4
NUMA node(s):        4
Model name:          Intel(R) Xeon(R) CPU E7-4870 @ 2.40GHz
````

---

## 2. `mpstat` – CPU Statistics

Shows per-CPU statistics like user, system, idle, and I/O wait.
Default reports CPU utilization since boot.

**Example:**

```bash
$ mpstat -P ALL
CPU   %usr   %sys   %idle
all   0.00   0.00   99.99
0     0.01   0.01   99.98
1     0.00   0.01   99.98
```

---

## 3. `sar` – CPU Activity Report

Reports CPU usage over time. Can display per-core statistics at intervals.

**Example:**

```bash
$ sar -P ALL 1 1
CPU    %user  %system  %iowait  %idle
all    11.69  4.71     0.69     82.90
0      35.00  6.00     0.00     59.00
1      22.00  5.00     0.00     73.00
```

# Memory Monitoring

## 1. `free` – Show Memory Usage
Displays information about total, used, free memory and swap space.

**Example:**
```bash
$ free -h
              total        used        free      shared  buff/cache   available
Mem:           992M         78M        324M        6.8M        590M        746M
Swap:          2.0G         70Mi       1.9G
````

---

## 2. `vmstat` – System Activity Report

Shows instant reports on processes, memory, swap, I/O, interrupts, and CPU usage. Can set intervals to monitor activity in real-time.

**Examples:**

```bash
$ vmstat -a
procs -----------memory---------- ---swap-- -----io---- -system-- ------cpu-----
r  b   swpd   free  inact active   si   so    bi    bo   in   cs us sy id wa st
1  0      0 332128  85800 505656    0    0     5    10   13   17  0  0 100  0  0

$ vmstat -f
11703 forks   # Total number of tasks created since last boot

$ vmstat -w 1 3   # Wide format, 3 samples at 1-second interval
$ vmstat -s       # Summary statistics
```

# Disk and I/O Monitoring Commands

## 1. `df` – Show Disk Space
Displays available and used disk space for all mounted filesystems.

**Example:**
```bash
$ df -h
````

---

## 2. `du` – Show Directory Disk Usage

Shows disk usage for a folder or files.

**Example:**

```bash
$ du -sh /etc/*
```

---

## 3. `lsof` – List Open Files

Lists all open files and associated processes. Can filter by command or network.

**Examples:**

```bash
$ lsof | less              # All open files
$ lsof /usr/bin/sshd        # Files for a specific process
$ lsof -c sshd              # Files for processes named 'sshd'
$ lsof -i                   # All Internet/network files
```

---

## 4. `vmstat` – System & Disk Statistics

Provides reports on processes, memory, I/O, and CPU activity.

**Examples:**

```bash
$ vmstat -d                  # Disk statistics
$ vmstat -p sda2             # Partition-specific stats
```

---

## 5. `iostat` – I/O Device Performance

Monitors system input/output device load and CPU usage.

**Examples:**

```bash
$ iostat -t                  # CPU and I/O with timestamp
$ iostat -d                  # Device-level I/O statistics
```

# Kernel and System Logging Commands

## 1. `dmesg` – Kernel Message Buffer
Prints kernel messages from the ring buffer, useful for hardware and boot diagnostics.

**Example:**
```bash
$ dmesg -T
[Sun Jul 12 15:43:37 2020] Initializing cgroup subsys cpuset
[Sun Jul 12 15:43:37 2020] Linux version 3.10.0-514.10.2.el7.x86_64 ...
````

---

## 2. `journalctl` – Systemd Journal Logs

Queries and displays logs collected by `systemd-journald`.

**Examples:**

```bash id="t85d2c"
$ journalctl -u httpd          # Logs for a specific service
$ journalctl -p err -b -x      # Errors since last boot, with explanations
$ journalctl -t dockerd         # Logs for a specific syslog identifier
$ journalctl --since "1 hour ago"  # Logs within a time range
$ journalctl --since "2020-03-26" --until "2020-03-27"
```

* `-u` → unit (service)
* `-p` → priority (log level)
* `-b` → since last boot
* `-x` → explanatory help texts
* `-t` → syslog identifier

---

## 3. Log Files in `/var/log/`

| File                                      | Description                                           |
| ----------------------------------------- | ----------------------------------------------------- |
| `/var/log/messages`                       | Global system messages (boot, cron, mail, kern, auth) |
| `/var/log/secure`                         | Authentication and authorization logs (e.g., sshd)    |
| `/var/log/cron`                           | Logs for cron jobs                                    |
| `/var/log/auth.log`                       | System authorization, user logins                     |
| `/var/log/maillog` or `/var/log/mail.log` | Mail server logs                                      |
| `/var/log/mail/`                          | Additional mail logs                                  |
| `/var/log/sa/`                            | Daily `sar` statistics collected by sysstat           |
