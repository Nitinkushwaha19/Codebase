# Linux Network Configuration and NSS

---

## 1. Configuring Network Interfaces

### Using `ifconfig` (legacy)
Legacy command to **configure, view, and manage network interfaces**. Can display interface status, assign IP addresses, and enable/disable interfaces. 
- **View interfaces:**
```bash
$ ifconfig           # Show active interfaces
$ ifconfig -a        # Show all interfaces (even down)
$ ifconfig eth0       # Show details of eth0
````

* **Activate/Deactivate interface:**

```bash
$ ifconfig eth0 up
$ ifconfig eth0 down
```

* **Set IP and netmask:**

```bash
$ ifconfig eth0 192.168.0.10 netmask 255.255.255.0
```

---

### Using `ip` (recommended modern tool)
Modern replacement for `ifconfig`. Used to **configure IP addresses, routes, and network interfaces**. Supports advanced networking features.

* **Assign static IP addresses:**

```bash
$ ip address add 192.168.2.223/24 dev eth1
$ ip addr show dev eth1
```

* **Add static routes:**

```bash
$ ip route add 192.0.2.1 via 10.0.0.2 dev eth0
$ ip route add 192.0.2.0/24 via 10.0.0.3 dev eth0
$ ip route
```

---

### Default Gateway

* Determined from `/etc/sysconfig/network` and interface-specific files (`ifcfg-ethX`) on Red Hat-based systems.
* **Restart network service:**

```bash
$ service network restart
```

**Example Red Hat configuration:**

```text
/etc/sysconfig/network-scripts/ifcfg-eth0
DEVICE=eth0
IPADDR=208.164.186.1
NETMASK=255.255.255.0
GATEWAY=192.168.1.1
ONBOOT=yes
BOOTPROTO=dhcp
```

**Example Debian/Ubuntu configuration:**

```text
/etc/network/interfaces
auto lo eth0
iface eth0 inet static
    address 192.168.0.10
    netmask 255.255.255.0
    gateway 192.168.0.1

/etc/hostname
myhost.local
```

## 2. Name Service Switch (NSS)

* Provides sources for system databases and name resolution.

* Sources include:

  * Local files → (`/etc/passwd`, `/etc/group`, `/etc/hosts`)
  * dns → Domain Name System
  * nis → Network Information Service
  * ldap → Lightweight Directory Access Protocol

* **Configuration file:** `/etc/nsswitch.conf`

**Example:**

```text
passwd:   files ldap
shadow:   files
group:    files ldap
hosts:    dns nis files
ethers:   files nis
netmasks: files nis
networks: files nis
protocols: files nis
rpc:      files nis
services: files
automount: files
aliases:  files
```

# Linux Network Diagnostic Commands


## 1. `ping`
- **Definition:** Sends ICMP echo requests to a host to check connectivity and measure round-trip time.  
- **Examples:**
```bash
$ ping google.com          # Continuous ping to google.com
$ ping -c 5 google.com     # Send 5 ping requests
````

---

## 2. `traceroute` / `tracepath`

* **Definition:** Shows the network route (hops) taken by packets to reach a destination host.
* **Examples:**

```bash
$ traceroute google.com
$ tracepath ya.ru
```

---

## 3. `host`

* **Definition:** Performs DNS lookups; resolves domain names to IP addresses and vice versa.
* **Examples:**

```bash
$ host ya.ru                # Get IP for ya.ru
$ host 213.180.204.8        # Reverse lookup
```

---

## 4. `dig`

* **Definition:** Flexible DNS lookup tool for querying DNS servers and performing reverse lookups.
* **Examples:**

```bash
$ dig ya.ru                 # DNS query
$ dig -x 213.180.204.8      # Reverse DNS query
```

---

## 5. `nslookup`

* **Definition:** Queries DNS servers to find IP addresses of domains or reverse lookups.
* **Examples:**

```bash
# Forward DNS lookup
$ nslookup epam.com

# Reverse DNS lookup
$ nslookup 174.128.60.201
```

---

## 6. `netstat`

* **Definition:** Displays network connections, routing tables, interface stats, listening ports, and more.
* **Examples:**

```bash
$ netstat -ant             # Show TCP connections
$ sudo netstat -nlpt        # Show listening ports with PID/program
```

---

## 7. `whois`

* **Definition:** Queries domain registration information from Network Information Centers (NICs).
* **Example:**

```bash
$ whois google.com
# Shows registrant, admin, creation/expiration date, etc.
```

---

## 8. `tcpdump`

* **Definition:** Captures and displays packets on a network interface matching a filter expression.
* **Examples:**

```bash
$ tcpdump -i eth0                       # Capture all traffic on eth0
$ tcpdump host 1.2.3.4                  # Traffic from/to 1.2.3.4
$ tcpdump port 80 -w capture_file       # Traffic on port 80, save to file
$ tcpdump 'src 10.0.2.4 and (dst port 3389 or 22)'  # Specific source/destination
```

# 🔥 Firewalld Overview

**Firewalld** is a dynamic firewall daemon for Linux that:  
- Supports **IPv4 & IPv6**, Ethernet bridges, IP sets.  
- Uses **zones** to assign trust levels to networks and interfaces.  
- Differentiates **runtime** (temporary) and **permanent** configurations.  
- Allows services and applications to add rules dynamically without losing connections.

---

## ⚔ Firewalld vs iptables

| Feature | iptables | firewalld |
|---------|----------|-----------|
| Config files | `/etc/sysconfig/iptables`, `/etc/sysconfig/ip6tables` | XML files in `/usr/lib/firewalld/` & `/etc/firewalld/` |
| Applying changes | Flushes all rules and reloads | Only applies differences dynamically |
| Effect on existing connections | May drop them | Existing connections are maintained |

---

## 🛡 Firewalld Zones

| Zone | Description |
|------|-------------|
| **drop** | All incoming connections are dropped; outgoing allowed. |
| **block** | Incoming connections rejected with ICMP message. |
| **public** | Public networks; allow selected incoming connections. |
| **external** | Gateway scenario; NAT masquerading enabled. |
| **internal** | Internal side of a gateway; trusted network with some services allowed. |
| **dmz** | Isolated computers; limited incoming connections. |
| **work** | Work machines; trust most computers, allow additional services. |
| **home** | Home network; trust most computers, allow more services. |
| **trusted** | Trust all machines; fully open (use sparingly). |

---

## ⚙ Firewalld Concepts & Commands

### 1. Masquerading
- **Definition:** Hides internal IP addresses when traffic passes through a firewall acting as a gateway. External hosts only see the firewall’s IP.  
- **Commands:**
```bash
# Enable masquerading on external zone
firewall-cmd --zone=external --add-masquerade

# Disable masquerading
firewall-cmd --zone=external --remove-masquerade

# Check if masquerading is active
firewall-cmd --zone=external --query-masquerade
````

* **Permanent changes:** Add `--permanent` and reload.

---

### 2. Port Forwarding

* **Definition:** Redirects incoming traffic from a specific port on the firewall to another port or internal host. Requires masquerading.
* **Command Example:**

```bash
# Forward port 22 to 3753 temporarily
firewall-cmd --zone=external --add-forward-port=port=22:proto=tcp:toport=3753
```

---

### 3. Panic Mode

* **Definition:** Temporarily drops **all network traffic**. Useful in emergencies.
* **Commands:**

```bash
firewall-cmd --panic-on   # Drop all packets
firewall-cmd --panic-off  # Resume normal traffic
```

---

### 4. Services

* **Definition:** Predefined rules for common services (like SSH, HTTP, FTP).
* **Commands:**

```bash
# List all predefined services
firewall-cmd --get-services

# View info about a service (ports, protocols, modules)
firewall-cmd --info-service=ftp
```

# 🔐 SSH, SCP, SFTP, and Remote Access

## 1. Protocols and Programs

| Protocol / Program | Definition / Use |
|-------------------|-----------------|
| **Telnet**        | Old protocol for remote console access; **not encrypted**. |
| **SSH**           | Secure Shell, encrypted protocol for remote login and command execution. |
| **WinSCP**        | GUI-based file manager for secure remote file copying via SSH. |
| **X-System**      | Client-server system for GUI applications; allows remote X11 forwarding. |
| **VNC**           | Virtual Network Computing; allows remote desktop access. |

---

## 2. SSH Key Generation

Generate a public/private key pair for passwordless authentication:

```bash
cd ~
ssh-keygen -t rsa
# Save to default: /home/user/.ssh/id_rsa
# Optionally add passphrase or leave empty
chmod 600 ~/.ssh/id_rsa
````

* **Private key:** `/home/user/.ssh/id_rsa` – Keep secret.
* **Public key:** `/home/user/.ssh/id_rsa.pub` – Copy to remote `~/.ssh/authorized_keys`.

---

## 3. SSH Command Options

| Option | Definition                                                               |
| ------ | ------------------------------------------------------------------------ |
| `-i`   | Specify private key file for authentication.                             |
| `-l`   | Specify remote login username.                                           |
| `-p`   | Specify port number on remote host.                                      |
| `-t`   | Force pseudo-terminal allocation (needed for sudo or screen-based apps). |
| `-v`   | Verbose mode; shows debug information.                                   |
| `-N`   | Do not execute a remote command; useful for port forwarding.             |

### Examples

```bash
# Basic SSH login
ssh -i ~/.ssh/lab.pem centos@remotehost

# Execute single command remotely
ssh centos@remotehost whoami

# Execute multiple commands
ssh centos@remotehost "whoami; pwd; ls"

# Sudo with pseudo-terminal
ssh -t centos@remotehost sudo ls /root

# Run local script remotely
ssh centos@remotehost < script.sh
```

> **Important:** SSH to `root` is forbidden by default. Configure `/etc/ssh/sshd_config` and restart sshd to allow.

---

## 4. SCP – Secure Copy

* Copies files over SSH.
* Syntax similar to SSH.

```bash
# Copy local file to remote host
scp examplefile centos@remotehost:/folder

# Copy remote file to local machine
scp centos@remotehost:/home/user/examplefile .

# Copy file between two remote hosts
scp remote1:/examplefile remote2:/home/user/
```

---

## 5. Bastion / SSH Proxy Command

* **Definition:** A bastion host is a jump server in the public subnet, used to access instances in private subnets.
* **SSH example using jump host:**

```bash
ssh -i ~/.ssh/lab.pem \
    -o "ProxyCommand ssh -W %h:%p -i jump_key.pem jumpuser@jumpbox_host" \
    centos@privatehost
```

---

## 6. SSH Config File

* Store host-specific options to simplify repeated SSH access.
* File: `~/.ssh/config`

```bash
touch ~/.ssh/config
chmod 600 ~/.ssh/config

# Example entry
Host targaryen
    HostName 192.168.1.10
    User daenerys
    Port 7654
    IdentityFile ~/.ssh/targaryen.key
```

* Connect with: `ssh targaryen` instead of full command.

---

## 7. SFTP – Secure File Transfer

* **Definition:** Securely transfer files between local and remote systems.
* **Supports:** SFTP, SCP, and FTP protocols over SSH.
* **Features:** Batch scripts, CLI, integrated editor, directory synchronization.

### SFTP Example:

```bash
sftp access@192.168.0.14
```

### Enforcing Chroot and Security in `sshd_config`

```text
Match User access
    ForceCommand internal-sftp
    PasswordAuthentication yes
    ChrootDirectory /var/sftp
    PermitTunnel no
    AllowAgentForwarding no
    AllowTcpForwarding no
    X11Forwarding no
```

* Apply changes: `systemctl restart sshd`
