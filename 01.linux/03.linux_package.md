# Software Package & Package Management System

## Software Package
A **software package** is a compressed archive that contains software files required for installation.  
It can be installed using a **package management system** or a standalone installer.

---

## Package Management System
A **package management system** is a collection of tools used to:
- Install software
- Upgrade software
- Configure packages
- Remove software automatically

---

## Linux Package Types

### 🔴 Red Hat–Based Distributions
Examples: RHEL, CentOS, Fedora

- Package Format: `*.rpm`
- Tools:
  - **rpm** → Low-level package manager
  - **yum** → High-level package manager (handles dependencies)

#### RPM (Red Hat Package Manager)
- Developed by Red Hat
- Used for installing, updating, and removing RPM packages

#### YUM (Yellowdog Updater Modified)
- Command-line package manager
- Automatically resolves dependencies
- Open-source utility under GNU GPL

---

### 🔵 Debian–Based Distributions
Examples: Debian, Ubuntu

- Package Format: `*.deb`
- Tools:
  - **dpkg** → Low-level installer
  - **apt / apt-get** → Advanced package manager

#### DEB Package
- Standard Debian software package format
- Name derived from **Debian**, founded by Ian Murdock

#### APT (Advanced Packaging Tool)
- Interface used to manage software installation and removal
- Handles dependencies automatically

---

## APT-GET Commands

| Command | Description |
|----------|-------------|
| `apt-get update` | Retrieve updated package lists |
| `apt-get upgrade` | Upgrade installed packages |
| `apt-get install pkg` | Install a package |
| `apt-get remove pkg` | Remove package |
| `apt-get purge pkg` | Remove package with configuration files |
| `apt-get check` | Check broken dependencies |

---

## APT-CACHE Commands

| Command | Description |
|----------|-------------|
| `apt-cache search` | Search packages |
| `apt-cache show` | Display package details |
| `apt-cache depends` | Show package dependencies |
| `apt-cache showpkg` | Show general package information |

# Linux Alternatives Command

`alternatives` manages **multiple versions of the same software** (e.g., Java, Python, Node.js) by creating and maintaining **symbolic links**.  
It allows easy switching between installed versions.



| Command | Purpose |
|---------|--------|
| `--install link name path priority` | Register a new version |
| `--config name` | Interactively select default version |
| `--set name path` | Manually set default version |
| `--auto name` | Automatically select highest priority version |
| `--display name` | Show all installed alternatives |
| `--remove name path` | Remove a version |

---

## Example

```bash
# Install Java alternative
alternatives --install /usr/bin/java java /usr/java/latest/bin/java 5

# Choose version
alternatives --config java

# Check version
java -version
```

# Linux Service Management

## 1️⃣ init.d (Legacy SysV Init System)
A legacy system in Unix-like OS that manages services using shell scripts stored in `/etc/init.d/`. Scripts define how to start, stop, or restart services during boot/shutdown.  

- **Location:** `/etc/init.d/`  
- **Purpose:** Manages services via shell scripts.  
- **Operation:** Scripts define how to start, stop, restart, or reload a service.  
- **Usage:** Scripts are executed during system boot and shutdown.  
- **Command Syntax:**
```bash
/etc/init.d/<service_name> <option>
````

Where `<option>` can be: `start`, `stop`, `restart`, `reload`, `force-reload`.


---

## 2️⃣ systemd (Modern Service Manager)

A modern system and service manager for Linux that uses declarative unit files to manage services, dependencies, and targets, providing better control, logging, and parallel booting.

* Includes tools like:

  * `systemd` → service manager
  * `journald` → logging
  * `logind` → user session management
  * `networkd` → network configuration
* Uses **targets** (.target) to group services instead of traditional runlevels (e.g., `multi-user.target`).

---

### systemd Unit Files

* **Locations:**

  * `/usr/lib/systemd/system/` → Package-provided units
  * `/run/systemd/system/` → Runtime units
  * `/etc/systemd/system/` → User-created units
* **Unit Types:** `service`, `socket`, `device`, `mount`, `swap`, `timer`, `target`, etc.

---

## Unit File Structure

Unit files have an **.ini-like format** with three main sections:

#### 1. [Unit]

* `Description=` → human-readable description
* `After=` / `Before=` → ordering dependencies
* `Requires=` / `Wants=` → strong/weak dependencies
* `Conflicts=` → units that cannot run together

#### 2. [Service]

* `Type=` → startup behavior (`simple`, `forking`, `oneshot`)
* `ExecStart=` → command to start service
* `ExecStop=` → command to stop service
* `ExecReload=` → command to reload service
* `Restart=` → behavior on failure
* `User=` / `Group=` → run service under specific user/group
* `WorkingDirectory=` → set working directory
* `Environment=` / `EnvironmentFile=` → environment variables

#### 3. [Install]

* `WantedBy=` → targets that want this service enabled
* `RequiredBy=` → targets that require it
* `Alias=` → additional names

---

### Example: Tomcat Service

`/etc/systemd/system/tomcat.service`

```ini
[Unit]
Description=Tomcat Application Server
After=network.target

[Service]
Type=forking
ExecStart=/apps/tomcat/bin/startup.sh
ExecStop=/apps/tomcat/bin/shutdown.sh
User=tomcat55
Group=staff

[Install]
WantedBy=multi-user.target
```

---

### systemd Commands

```bash
systemctl start tomcat.service
systemctl stop tomcat.service
systemctl restart tomcat.service
systemctl reload tomcat.service
systemctl status tomcat.service
systemctl is-active tomcat.service
systemctl list-units --type service --all
```

# Linux Cron & Crontab (Short Overview)

## 1️⃣ Crond (Cron Daemon)
`crond` is the background daemon that runs scheduled commands (cron jobs) on Linux. It continuously checks crontab files and executes commands at their scheduled times.  
- **Start Cron:** From `/etc/rc.d/init.d` or `/etc/init.d`.
- **Crontab Files Locations:**
  - **User crontabs:**  
    - Red Hat/CentOS: `/var/spool/cron/`  
    - Debian/Ubuntu: `/var/spool/cron/crontabs/`
  - **System-wide crontabs:** `/etc/crontab`, `/etc/cron.d/`
  - Scripts in `/etc/cron.{hourly,daily,weekly,monthly}` also run automatically.

- **Operation:**  
  Cron reads crontab files and executes commands at the scheduled times. Output is emailed to the user or can be sent to syslog (`-s` option).

---

## 2️⃣ Crontab Commands
- **Edit current user crontab:**
```bash
crontab -e
````

* **Edit another user crontab:**

```bash
crontab -u username -e
```

* **List crontab jobs:**

```bash
crontab -l
```

* **Remove crontab jobs:**

```bash
crontab -r
```

---

## 3️⃣ Crontab Syntax

A crontab line has **5 time fields** followed by the command:

```bash
* * * * * command_to_run
| | | | |
| | | | +--- Day of week (0-7, 0 or 7 = Sunday)
| | | +----- Month (1-12)
| | +------- Day of month (1-31)
| +--------- Hour (0-23)
+----------- Minute (0-59)
```

* Use `,` for multiple values, `-` for range, `*/n` for intervals.

---

## 4️⃣ Examples

| Schedule              | Cron Line                                                   | Description                 |
| --------------------- | ----------------------------------------------------------- | --------------------------- |
| Daily at 2 AM         | `0 2 * * * /bin/sh backup.sh`                               | Run backup daily            |
| Twice a day           | `0 5,17 * * * /scripts/script.sh`                           | Run at 5 AM & 5 PM          |
| Every minute          | `* * * * * /scripts/script.sh`                              | Run every minute            |
| Every Sunday 5 PM     | `0 17 * * sun /scripts/script.sh`                           | Run weekly                  |
| Every 10 minutes      | `*/10 * * * * /scripts/monitor.sh`                          | Interval job                |
| Specific months       | `* * jan,may,aug * /scripts/script.sh`                      | Run in Jan, May, Aug        |
| Multiple commands     | `* * * * * /scripts/script1.sh; /scripts/script2.sh`        | Run two scripts in one cron |
| First Sunday of month | `0 2 * * sun [ $(date +\%d) -le 07 ] && /scripts/script.sh` | Conditional execution       |
