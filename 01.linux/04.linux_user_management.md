# UNIX Users and Groups

## Users

Each user has the following attributes:

- **User name** – Unique identifier for logging into the system.
- **Encrypted password** – User’s password stored in hashed form; `x` indicates it’s in the shadow file.
- **UID (User Identifier)** – Unique numeric ID assigned to each user.
- **GID (Group Identifier)** – Primary group ID associated with the user.
- **Full name / Description** – Human-readable description of the user.
- **Home directory** – Default directory where the user’s personal files are stored.
- **Shell** – Command-line interface assigned to the user (e.g., `/bin/bash`).
- **Expiration date** – Date when the user account will expire (optional).

**Default users** include `root`, `daemon`, `bin`, `sys`, `adm`, `lp`, `uucp`, `nobody`, etc.

**User Commands:**

| Command | Description |
|---------|-------------|
| `su` | Switch to another user (requires that user’s password). |
| `sudo` | Run commands as another user (usually root) using your own password. |
| `useradd` | Create a new user account. |
| `userdel` | Delete a user account (`-r` removes home directory). |
| `usermod` | Modify user account settings (home directory, shell, groups, etc.). |
| `passwd` | Change a user’s password. |
| `finger` | Display detailed information about a user. |

---

## Groups

Each group has the following attributes:

- **Group name** – Unique identifier for the group.
- **Encrypted password** – Optional password for the group.
- **GID (Group Identifier)** – Unique numeric ID for the group.
- **Members** – List of usernames that belong to the group.

**Group Commands:**

| Command | Description |
|---------|-------------|
| `groupadd` | Create a new group. |
| `groupdel` | Delete an existing group. |
| `groupmod` | Modify an existing group (name, GID, etc.). |
| `groups` | Display the groups a user belongs to. |

---

**Notes:**
- `root` is the superuser with unlimited rights.  
- Every user belongs to at least one group.  
- Prefer using `sudo` for administrative tasks instead of logging in as `root`.

# UNIX File Permissions, Ownership, and Special Attributes

## File and Directory Ownership

- Files and directories are assigned:
  - **User owner** (UID)
  - **Group owner** (GID)
- **Commands:**
  - `chown user /path` – Change the owner of a file or directory.
  - `chgrp group /path` – Change the group owner of a file or directory.
- `ls -l` – View file ownership and permissions.

---

## Access Permissions

- Permissions are granted to **user (owner)**, **group**, and **other (world)**.
- Types of access:
  - **r** – Read (4)
  - **w** – Write (2)
  - **x** – Execute (1)
- **Octal representation example:**
  - `rwxr-x---` → 7 5 0  
    - Owner: `r+w+x = 7`
    - Group: `r+x = 5`
    - Other: `--- = 0`
- **Commands:**
  - `chmod g=rw file` – Give read/write to group.
  - `chmod 755 file` – Set `rwxr-xr-x` permissions using octal.
  - `chmod o-r,g+w file` – Remove read from others, add write to group.

---

## Sticky Bit

- Used on **shared directories** to prevent users from deleting others' files.
- Example: `/tmp` with sticky bit prevents one user from deleting another's files even if directory is `777`.
- **Commands:**
  - `chmod +t directory` – Set sticky bit.
  - `chmod -t file` – Remove sticky bit.
  - Octal: `chmod 1700 directory`

---

## SUID and SGID

- **SUID (Set User ID):**
  - Executes a program with the **owner’s UID** instead of the caller’s.
  - `chmod 4555 /path/to/file` – Set SUID.
  - Shown as `s` in owner's execute field (`rwsr-xr-x`).

- **SGID (Set Group ID):**
  - Executes a program with the **file’s group**.
  - `chmod 2555 /path/to/folder` – Set SGID.
  - Shown as `s` in group execute field (`rwxr-sr-x`).

---

## File Attributes

- **lsattr** – Lists special attributes of files.
  - Example: `lsattr /path/to/file`  

- **chattr** – Change file system attributes:
  - `i` – Immutable: file cannot be modified (`chattr +i file`)
  - `A` – No access time: access does not update last-access time
  - `a` – Append-only: data can only be added, cannot delete/modify existing
  - `s` – Secure deletion: overwritten with zeroes when deleted
 
  **Example**
  ```bash
    # Make a file immutable (cannot modify or delete)
    $ sudo chattr +i testfile.txt

    # Verify using lsattr
    $ lsattr testfile.txt
    ----i--------e-- testfile.txt
    ```

# SELinux (Security Enhanced Linux)

SELinux is an advanced **access control mechanism** in Linux that implements **MAC (Mandatory Access Control)** on top of the default **DAC (Discretionary Access Control)**.

---

## SELinux Modes

| Mode       | Description |
|------------|-------------|
| **Enforcing** | SELinux enforces policies; unauthorized access is denied and logged. |
| **Permissive** | SELinux logs policy violations but does not deny access; useful for testing. |
| **Disabled** | SELinux is completely turned off. |

**Check current mode:**

```bash
$ getenforce        # Shows: Enforcing / Permissive / Disabled
$ sestatus          # Shows detailed SELinux status
````

---

## SELinux Configuration

* Configuration file: `/etc/selinux/config`
* Important parameters:

```text
SELINUX=disabled     # enforcing, permissive, or disabled
SELINUXTYPE=targeted # targeted, minimum, or mls
```

**Change mode:**

```bash
# Enable SELinux in permissive mode
$ vi /etc/sysconfig/selinux
SELINUX=permissive
$ reboot

# Enable SELinux enforcing mode
$ vi /etc/sysconfig/selinux
SELINUX=enforcing
$ reboot

# Disable SELinux
$ vi /etc/sysconfig/selinux
SELINUX=disabled
$ reboot
```

---

## SELinux Policies

* List currently loaded policy modules:

```bash
$ semodule -l | less
```

**Example output:**

```
abrt 1.2.0
apache 2.4.0
aide 1.6.1
...
```

---

## SELinux Booleans

* Booleans control optional SELinux features dynamically without changing policies.
* List all Booleans:

```bash
$ semanage boolean -l | less
```

* Check a specific Boolean:

```bash
$ getsebool ftpd_anon_write
ftpd_anon_write --> off
```

* Enable/disable a Boolean:

```bash
$ setsebool ftpd_anon_write on
$ getsebool ftpd_anon_write
ftpd_anon_write --> on
```
