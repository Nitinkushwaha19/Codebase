

# Docker Logging - Short Notes

## 1. WHAT IS DOCKER LOGGING?

**Definition:** Captures STDOUT and STDERR from containers for monitoring, debugging, and compliance.

**Why Important:**
- Troubleshooting application issues
- Performance monitoring
- Security auditing
- Compliance requirements

---

## 2. HOW DOCKER HANDLES LOGS

**Streams Captured:**
- **STDOUT** - Normal output
- **STDERR** - Error messages

**Default:**
- Driver: `json-file`
- Location: `/var/lib/docker/containers/<id>/<id>-json.log`

---

## 3. LOG CATEGORIES

### Infrastructure Logs
**For:** Operators, DevOps teams
**Contains:** Docker daemon logs, container lifecycle, resource usage

### Application Logs  
**For:** Developers, support teams
**Contains:** App events, errors, user activity, business logic

---

## 4. LOG DRIVERS

| Driver | Purpose | Example |
|--------|---------|---------|
| **json-file** | Default, local files | `docker run --log-driver json-file nginx` |
| **syslog** | System logging | `docker run --log-driver syslog nginx` |
| **journald** | systemd journal | `docker run --log-driver journald nginx` |
| **fluentd** | Centralized logging | `docker run --log-driver fluentd nginx` |
| **awslogs** | AWS CloudWatch | `docker run --log-driver awslogs nginx` |

---

## 5. CONFIGURATION

### Global Config
**File:** `/etc/docker/daemon.json`
```json
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  }
}
```

### Per-Container Config
```bash
docker run \
  --log-driver json-file \
  --log-opt max-size=10m \
  --log-opt max-file=3 \
  nginx
```

---

## 6. COMMON COMMANDS

```bash
# View logs
docker logs <container>

# Follow logs real-time
docker logs -f <container>

# Last 50 lines with timestamps
docker logs -t --tail 50 <container>

# Logs since 1 hour ago
docker logs --since "1h" <container>

# Docker daemon logs
sudo journalctl -u docker.service
```

---

## 7. PRACTICAL EXAMPLES

### Basic Logging Setup
```bash
# Development
docker run -d \
  --name webapp \
  --log-opt max-size=5m \
  nginx

# Production with rotation
docker run -d \
  --name prod-app \
  --log-driver json-file \
  --log-opt max-size=10m \
  --log-opt max-file=5 \
  myapp
```

### Docker Compose
```yaml
services:
  web:
    image: nginx
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"
```

---

## 8. BEST PRACTICES

- ✅ **Use log rotation** (`max-size`, `max-file`)
- ✅ **Centralize logs** in production
- ✅ **Include timestamps** (`-t` flag)
- ✅ **Monitor log disk usage**
- ✅ **Use structured logging** (JSON format)
- ✅ **Don't log sensitive data**

---

## 9. QUICK REFERENCE

```bash
# Essential commands
docker logs -f --tail 100 <container>    # Follow last 100 lines
docker logs --since "2h" <container>     # Last 2 hours
docker inspect <container> | grep LogPath # Find log file location

# Log cleanup
echo "" > $(docker inspect --format='{{.LogPath}}' <container>)
```
