# 📌 Introduction to Docker

Docker is a **containerization platform** that packages applications and their dependencies into lightweight, portable containers that run consistently across environments.

A container includes:

* Application code
* Runtime
* Libraries
* System dependencies

---

# ❓ Why Docker? (Problems It Solves)

Before Docker:

* “Works on my machine” problem
* Dependency conflicts
* Complex deployments
* Heavy virtual machines

Docker solves this by:

* Creating consistent environments
* Making deployments predictable
* Reducing infrastructure overhead
* Supporting microservices architecture

---

# 🖥 Virtual Machines vs Containers

| Virtual Machine     | Docker Container  |
| ------------------- | ----------------- |
| Includes full OS    | Shares host OS    |
| Heavy (GBs)         | Lightweight (MBs) |
| Slow boot time      | Starts instantly  |
| Requires hypervisor | Uses OS kernel    |

Containers are more efficient because they share the host operating system kernel.

# ⚙ Docker Installation (Ubuntu Example)

```bash
sudo apt update
```

### Install Docker

```bash
sudo apt install docker.io
```


# 📦 Docker Images

A Docker image is a **read-only blueprint** used to create containers.

### List Images

```bash
docker images
```

### Pull an Image

```bash
docker pull nginx
```

### Remove an Image

```bash
docker rmi nginx
```

### Inspect an Image

```bash
docker inspect nginx
```

---

# 🚀 Docker Containers

A container is a **running instance of an image**.

### Run Container

```bash
docker run nginx
```

### Run in Background

```bash
docker run -d nginx
```

### Run with Port Mapping

```bash
docker run -d -p 8080:80 nginx
```

Access in browser:

```
http://localhost:8080
```

### List Containers

```bash
docker ps
docker ps -a
```

### Stop Container

```bash
docker stop <container_id>
```

### Remove Container

```bash
docker rm <container_id>
```

---

# 📁 Docker Volumes (Persistent Storage)

Containers are **ephemeral**, meaning data is lost when they are removed.
Volumes allow data to persist outside the container lifecycle.

### Create Volume

```bash
docker volume create myvolume
```

### Use Volume

```bash
docker run -v myvolume:/app/data nginx
```

### Bind Mount (Local Folder)

```bash
docker run -v $(pwd):/app nginx
```

Bind mounts directly connect a host directory to the container and are commonly used in development.

---

# 🌐 Docker Networking

Docker provides built-in networking to allow containers to communicate securely.

### Types of Networks

* **Bridge** (default) – Containers communicate within the same host
* **Host** – Container shares host network
* **None** – No networking
* **Custom** – User-defined isolated networks

### Create Network

```bash
docker network create mynetwork
```

### Run Container in a Network

```bash
docker run -d --network mynetwork nginx
```

# 🛠 Dockerfile

A **Dockerfile** is a text file that contains instructions to build a custom Docker image.
It automates image creation and ensures consistent builds across environments.

---

## 🔹 Basic Dockerfile Example

```dockerfile
FROM nginx:latest
COPY . /usr/share/nginx/html
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

---

## 🔹 Build Image

```bash
docker build -t mynginx .
```

Builds an image named `mynginx` using the current directory as the build context.

---

# 🔹 Common Dockerfile Instructions

### `FROM`

Defines the base image used to build the container image.

### `WORKDIR`

Sets the working directory for subsequent instructions inside the container.

### `COPY`

Copies files from the host machine into the image during build time.

### `RUN`

Executes commands while building the image (e.g., installing dependencies).

### `EXPOSE`

Documents which port the container listens on at runtime.

### `CMD`

Specifies the default command executed when the container starts.

### `ENTRYPOINT`

Defines the main executable of the container.

# 📦 Docker Compose

Docker Compose is a tool for defining and managing multi-container applications using a YAML file.
It allows you to start multiple services with a single command.

---

## 🔹 docker-compose.yml Example

```yaml
version: "3.8"

services:
  web:
    image: nginx
    ports:
      - "8080:80"

  db:
    image: mysql
    environment:
      MYSQL_ROOT_PASSWORD: root
```

---

# 🔹 Docker Compose Parameters

### `version`

Specifies the Compose file format version.

### `services`

Defines the list of containers that make up the application.

### `web` / `db`

Service names used for container identification and internal networking.

### `image`

Specifies the Docker image used to create the container.

### `ports`

Maps host ports to container ports (`host:container` format).

### `environment`

Sets environment variables inside the container.

---

# 🔹 Common Docker Compose Commands

### Start Services

```bash
docker-compose up -d
```

Creates and runs containers in detached mode.

---

### Stop Services

```bash
docker-compose down
```

Stops and removes containers, networks, and volumes.

---

### View Running Services

```bash
docker-compose ps
```

Displays the status of services.
