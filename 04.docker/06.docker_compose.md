## 1. WHAT IS DOCKER COMPOSE?

**Definition:** Docker Compose is a tool for defining and running multi-container Docker applications using a YAML configuration file.

**Purpose:** Simplifies managing multiple containers that need to work together instead of using complex `docker run` commands with multiple options.

**Default File:** `./docker-compose.yml`

---

## 2. WHY USE DOCKER COMPOSE?

### Problems It Solves:
- Managing multiple `docker run` commands with many options (-v, -it, -e, --restart)
- Creating networks and volumes manually
- Coordinating container startup order
- Managing container dependencies

### Use Cases:
- **Development environments**
- **Automated testing environments** 
- **Single host deployments**

### Key Features:
- Multiple isolated environments on single host
- Preserve volume data when containers recreated
- Only recreate containers that changed
- Variables for different environments

---

## 3. DOCKER COMPOSE FILE STRUCTURE

**Format:** YAML file with these sections:

```yaml
version: '3.8'          # Compose file version
services:               # Container definitions
networks:               # Custom networks
volumes:                # Named volumes
configs:                # Configuration files
```

**Important:** Must specify exact version (no backward compatibility)

---

## 4. BUILDING DOCKER COMPOSE CONFIGURATION

### Step 1: Split App into Services
```yaml
services:
  client:     # Frontend web app
  server:     # Backend API
  database:   # Database
```

### Step 2: Pull or Build Images
```yaml
services:
  # Using public image
  database:
    image: "postgres:13"
  
  # Building from Dockerfile
  server:
    build: ./server-directory
```

### Step 3: Environment Variables
```yaml
services:
  database:
    image: postgres:13
    environment:
      POSTGRES_USER: myuser
      POSTGRES_DB: mydb
      POSTGRES_PASSWORD: secret
```

**Alternative:** Use `.env` file in same directory
```bash
# .env file
POSTGRES_USER=myuser
POSTGRES_DB=mydb
```

### Step 4: Networking
```yaml
services:
  web:
    ports:
      - "8080:80"    # host:container
  
  api:
    # No ports = internal only
    # Accessible via service name: api:3000
```

### Step 5: Volumes
```yaml
services:
  database:
    volumes:
      - db_data:/var/lib/postgresql/data

volumes:
  db_data:    # Named volume declaration
```

### Step 6: Build and Run
```bash
# Build services
docker-compose build

# Start all services
docker-compose up

# Start
 in background
docker-compose up -d
```

## 5. COMPLETE EXAMPLE

```yaml
version: '3.8'

services:
  # Database service
  database:
    image: postgres:13
    environment:
      POSTGRES_USER: myuser
      POSTGRES_DB: myapp
      POSTGRES_PASSWORD: secret
    volumes:
      - db_data:/var/lib/postgresql/data
    networks:
      - backend

  # API service
  api:
    build: ./api
    environment:
      DATABASE_URL: postgresql://myuser:secret@database:5432/myapp
    depends_on:
      - database
    networks:
      - backend
      - frontend

  # Web frontend
  web:
    build: ./web
    ports:
      - "8080:80"
    depends_on:
      - api
    networks:
      - frontend

networks:
  frontend:
  backend:

volumes:
  db_data:
```

## 6. ESSENTIAL COMMANDS

### Basic Operations
```bash
# Start all services
docker-compose up

# Start in background
docker-compose up -d

# Stop all services
docker-compose down

# View running services
docker-compose ps

# View logs
docker-compose logs

# Follow logs
docker-compose logs -f
```

### Service Management
```bash
# Build specific service
docker-compose build api

# Start specific service
docker-compose up api

# Restart service
docker-compose restart api

# Stop specific service
docker-compose stop api

# Execute command in running container
docker-compose exec api bash

# View images
docker-compose images
```

### Advanced Operations
```bash
# Scale service (multiple instances)
docker-compose up --scale api=3

# Force recreate containers
docker-compose up --force-recreate

# Remove stopped containers
docker-compose rm

# Pull latest images
docker-compose pull
```

## 7. PRACTICAL EXAMPLES

### Web App with Database
```yaml
version: '3.8'
services:
  web:
    image: nginx
    ports:
      - "80:80"
    volumes:
      - ./html:/usr/share/nginx/html
  
  db:
    image: mysql:8
    environment:
      MYSQL_ROOT_PASSWORD: rootpass
      MYSQL_DATABASE: webapp
    volumes:
      - mysql_data:/var/lib/mysql

volumes:
  mysql_data:
```

### Development Environment
```yaml
version: '3.8'
services:
  app:
    build: .
    volumes:
      - .:/app          # Mount source code
      - /app/node_modules  # Exclude node_modules
    ports:
      - "3000:3000"
    environment:
      NODE_ENV: development
    command: npm run dev
```

## 8. BEST PRACTICES

- ✅ **Use specific image tags** (not `latest`)
- ✅ **Use `.env` files** for environment variables
- ✅ **Use named volumes** for persistent data
- ✅ **Define custom networks** for service isolation
- ✅ **Use `depends_on`** for startup order
- ✅ **Keep compose files in version control**
- ✅ **Use health checks** for service readiness

    build: ./path

    depends_on:

  network_name:

volumes:
  volume_name:
