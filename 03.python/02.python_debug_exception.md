
# Python Exceptions

## Definition
An **exception** is an error that occurs during the execution of a program.  
It interrupts the normal flow of the program and transfers control to an **exception handler**.

Exceptions usually occur when something unexpected happens, such as:

- Division by zero
- Invalid input
- File not found
- Network errors

Exception handling allows a program to **detect errors and handle them without crashing**.

---

# Exception Handling in Python

Python uses the following keywords for exception handling:

- `try`
- `except`
- `else`
- `finally`

### Basic Syntax

```python
try:
    # Code that may cause an exception
except Exception1:
    # Executes if Exception1 occurs
except Exception2:
    # Executes if Exception2 occurs
else:
    # Executes if no exception occurs
finally:
    # Always executes
````

---

# Example

```python
try:
    num = int(input("Enter a number: "))
    result = 10 / num
except ZeroDivisionError:
    print("Cannot divide by zero")
except ValueError:
    print("Invalid input")
else:
    print("Result:", result)
finally:
    print("Execution completed")
```

---

# Handling Multiple Exceptions

Multiple exceptions can be handled in a single block.

```python
try:
    # Code that may cause error
except (TypeError, ValueError) as exc:
    print("Error occurred:", exc)
else:
    print("No exception occurred")
```

---

# Exception Argument Example

Exceptions can return information about the error.

```python
def temp_convert(var):
    try:
        return int(var)
    except ValueError as argument:
        print("The argument does not contain numbers:", argument)

temp_convert("xyz")
```

### Output

```
The argument does not contain numbers: invalid literal for int() with base 10: 'xyz'
```

---

# Raising an Exception

The `raise` keyword is used to **manually trigger an exception** when a specific condition occurs.

### Syntax

```python
raise Exception("Error message")
```

### Example

```python
def functionName(level):
    if level < 1:
        raise Exception("Invalid level! %s" % level)
```

---
# Using Raise with Exception Handling

```python
try:
    functionName(0)
except Exception as e:
    print("Exception occurred:", e)
else:
    print("Program executed successfully")
```

---

# Custom Exceptions

Python allows developers to **create their own exception classes**.

### Example

```python
class Networkerror(RuntimeError):
    def __init__(self, message):
        self.message = message

try:
    raise Networkerror("Bad hostname")
except Networkerror as e:
    print(e.message)
```

### Output

```
Bad hostname
```

# Debugging in Python

## Definition
**Debugging** is the process of **finding, analyzing, and fixing errors (bugs)** in a program.  
In Python, debugging can be done using:

- Debuggers
- Logging
- Print statements
- Error traces (stack traces)

Debugging helps developers understand **why a program is not behaving as expected**.

---

# Logging

## Definition
**Logging** is a technique used to **record information about a program’s execution**.  
It helps developers track program flow, detect errors, and analyze application behavior.

Logs can provide details such as:

- Program execution steps
- Errors and warnings
- User activity
- Application performance

Python provides a built-in **logging module** for implementing logging in applications.

---

# Logging Benefits

- Helps debug errors easily
- Tracks program flow
- Stores runtime information
- Helps analyze application performance
- Useful for monitoring and scaling applications

---

# Logging with syslog

Python can write log messages to the **system log (syslog)**.

### Example

```python
# debug.py
from syslog import syslog

syslog('This is a debug message.')
````

Run the program:

```bash
python debug.py
```

View logs:

```bash
tail /var/log/syslog
```

---

# Python Logging Module

Python provides a **powerful logging module** in its standard library.

It supports multiple logging levels.

### Logging Levels

| Level    | Description                                 |
| -------- | ------------------------------------------- |
| DEBUG    | Detailed diagnostic information             |
| INFO     | General information about program execution |
| WARNING  | Indicates a potential problem               |
| ERROR    | Indicates a serious problem                 |
| CRITICAL | Indicates a very serious error              |

---

# Logging Example

```python
import logging

logging.debug('This is a debug message')
logging.info('This is an info message')
logging.warning('This is a warning message')
logging.error('This is an error message')
logging.critical('This is a critical message')
```

### Example Output

```
WARNING:root:This is a warning message
ERROR:root:This is an error message
CRITICAL:root:This is a critical message
```

---

# Setting Logging Level

You can control which log messages appear using `basicConfig`.

### Example

```python
import logging

logging.basicConfig(level=logging.INFO)

logging.info('This will get logged')
logging.debug('This will not get logged')
```

Only messages with level **INFO or higher** will be displayed.

---

# Logging to a File

Logs can also be stored in a file.

### Example

```python
import logging

logging.basicConfig(
    filename='app.log',
    filemode='w',
    format='%(name)s - %(levelname)s - %(message)s'
)

logging.error('This will get logged to a file')
```

### Output in File

```
root - ERROR - This will get logged to a file
```

---

# Advanced Logging Configuration

Complex logging configurations can be defined in a **configuration file**.

### Example YAML Configuration

```yaml
version: 1
formatters:
  simple:
    format: '%(asctime)s - %(name)s - %(levelname)s - %(message)s'

handlers:
  console:
    class: logging.StreamHandler
    level: DEBUG
    formatter: simple
    stream: ext://sys.stdout

root:
  level: DEBUG
  handlers: [console]
```

---

# Loading Configuration in Python

```python
import logging
import logging.config
import yaml

with open('config.yaml', 'r') as f:
    config = yaml.safe_load(f.read())
    logging.config.dictConfig(config)

logger = logging.getLogger("simpleExample")

logger.debug('This is a debug message')
```

# Python Debugger (pdb)

Python provides a **built-in debugger module** called `pdb` that allows you to **step through code, inspect variables, and understand program flow**.

It is useful in environments **without a GUI**, such as Docker containers or remote servers.

---

## 1️⃣ Example Script for Debugging

```python id="lh6p3v"
# debug.py
a = "aaa"
b = "bbb"
c = "ccc"
final = a + b + c
print(final)
````

---

## 2️⃣ Running the Debugger

Use the following command to start debugging:

```bash id="fdl1sm"
python -m pdb debug.py
```

This will enter **interactive pdb mode**, stopping at the first line of the script:

```
> /home/user/debug.py(1)<module>()
-> a = "aaa"
(Pdb)
```

---

## 3️⃣ Common pdb Commands

| Command           | Description                                                                          |
| ----------------- | ------------------------------------------------------------------------------------ |
| `n` or `next`     | Continue to the **next line** in the current function.                               |
| `s` or `step`     | Step **into a function** if a function call is encountered.                          |
| `c` or `continue` | Continue execution until the **next breakpoint**.                                    |
| `p` or `print`    | Evaluate and **print a variable or expression**. Example: `p a,b` → `('aaa', 'bbb')` |
| `q` or `quit`     | Quit the debugger and **abort program execution**.                                   |

---

### 4️⃣ Example Usage

```text
(Pdb) n
> /home/user/debug.py(2)<module>()
-> b = "bbb"
(Pdb) s
> /home/user/debug.py(3)<module>()
-> c = "ccc"
(Pdb) p a,b
('aaa', 'bbb')
(Pdb) c
aaabbbccc
(Pdb) q
```

---

## 5️⃣ Setting Breakpoints with `pdb.set_trace()`

You can insert a **breakpoint** directly in your code:

```python id="jpxq6v"
import pdb

a = "aaa"
pdb.set_trace()  # Program stops here
b = "bbb"
c = "ccc"
final = a + b + c
print(final)
```

* When the interpreter reaches `pdb.set_trace()`, the program **pauses**, and you enter interactive pdb mode.
* You can then use the commands `n`, `s`, `p`, `c`, and `q` to inspect and control execution.

---

### 6️⃣ Benefits of Using pdb

* Step through code line by line
* Inspect variables at runtime
* Diagnose logical errors
* Set breakpoints anywhere in the code
* Useful in CLI-only environments

# Jinja2 Configuration Templating

**Jinja2** is a Python templating engine used to **dynamically generate configuration files**.

It is widely used in automation tools like:
- Ansible
- Flask
- SaltStack

---

## 1️⃣ Installation
```bash
pip install Jinja2 pyyaml
```

---

## 2️⃣ Template Syntax

| Tag         | Purpose           | Example                                |
| ----------- | ----------------- | -------------------------------------- |
| `{{ var }}` | Variable          | `{{ servername }}`                     |
| `{% %}`     | Control (if, for) | `{% if serveradmin %} ... {% endif %}` |
| `{# #}`     | Comment           | `{# This is a comment #}`              |

---

## 3️⃣ Example: Render Template

```python
import yaml
from jinja2 import Template

with open('data.yml') as f:
    data = yaml.safe_load(f)

with open('vhosts.j2') as f:
    template = Template(f.read())

conf = template.render(data)

with open('vhosts.conf', 'w') as f:
    f.write(conf)
```

---

## 4️⃣ Loops & Conditionals

```jinja2
{% for vhost in apache_vhosts %}
<VirtualHost *:80>
  ServerName {{ vhost.servername }}
  DocumentRoot {{ vhost.documentroot }}
  {% if vhost.serveradmin %}
  ServerAdmin {{ vhost.serveradmin }}
  {% endif %}
</VirtualHost>
{% endfor %}
```

`data.yml` example:

```yaml
apache_vhosts:
  - servername: www.domain1.tld
    documentroot: /www/domain1
    serveradmin: admin@domain1
  - servername: www.domain2.tld
    documentroot: /www/domain2
```
