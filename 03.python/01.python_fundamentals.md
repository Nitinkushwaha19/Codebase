# Python Basics – Installation, Virtual Environments & Language Syntax

## 1. Python Definition
Python is a **high-level, interpreted, general-purpose programming language** known for its **simple syntax and readability**.  
It is widely used for **web development, automation, data science, artificial intelligence, scripting, and software development**.

Created by **Guido van Rossum** and first released in **1991**.

---

# Python Features

- **Easy to Learn** – Simple and readable syntax.
- **Interpreted Language** – Code runs line by line without compilation.
- **Platform Independent** – Works on Windows, Linux, and macOS.
- **Object-Oriented** – Supports classes and objects.
- **Large Standard Library** – Many built-in modules available.
- **Open Source** – Free to use and modify.
- **Extensive Community Support** – Large number of developers and libraries.

---

# Virtual Environments

A **virtual environment** is an isolated Python environment used to **manage project dependencies separately**.

### Why Use Virtual Environments

* Avoid package conflicts
* Maintain project-specific libraries
* Easy dependency management

### Create Virtual Environment

```bash
python -m venv myenv
```

### Activate Virtual Environment

Linux / macOS

```bash
source myenv/bin/activate
```

Windows

```bash
myenv\Scripts\activate
```

### Deactivate Environment

```bash
deactivate
```

# Pyenv

## Definition
**Pyenv** is a tool used to **install, manage, and switch between multiple Python versions** on a single machine.  
It helps developers work on different projects that require **different Python versions** without conflicts.

---

## Features

- Install multiple Python versions
- Switch between Python versions easily
- Create and manage virtual environments
- Set global, local, or shell-specific Python versions
- Works well with package managers like **pip**

---

## Installation

Install pyenv using the following command:

```bash
curl -L https://raw.github.com/yyuu/pyenv-installer/master/bin/pyenv-installer | bash
````

More details:
[https://github.com/yyuu/pyenv](https://github.com/yyuu/pyenv)

---

## Important Pyenv Commands

### List Available Python Versions

```bash
pyenv install -l
```

Shows all Python versions available for installation.

---

### Install a Python Version

```bash
pyenv install <version>
```

Example:

```bash
pyenv install 3.8.11
```

---

### List Installed Versions

```bash
pyenv versions
```

Displays all Python versions installed on your system.

---

### Set Global Python Version

```bash
pyenv global <version>
```

Example:

```bash
pyenv global 3.8.11
```

Sets the **default Python version** for the entire system.

---

### Set Local Python Version

```bash
pyenv local <version>
```

This sets the Python version **only for the current project directory**.

---

## Virtual Environment with Pyenv

### Create Virtual Environment

```bash
pyenv virtualenv <version> <env_name>
```

Example:

```bash
pyenv virtualenv 3.9.6 my-project
```

---

### Activate Virtual Environment

```bash
pyenv activate <env_name>
```

---

### Deactivate Virtual Environment

```bash
pyenv deactivate
```

---

### Remove Virtual Environment

```bash
pyenv uninstall <env_name>
```

---

## Example Workflow

```bash
pyenv install 3.8.11
pyenv global 3.8.11
pyenv virtualenv 3.8.11 flaskage
```

# Python Advanced Concepts

## 1. Scopes

### Definition
**Scope** refers to the **region of a program where a variable is accessible**.

Python determines the scope of variables using the **LEGB rule**:

- **L – Local** → Variables defined inside a function
- **E – Enclosing** → Variables in outer functions (nested functions)
- **G – Global** → Variables defined at the top level of a script
- **B – Built-in** → Predefined names in Python (e.g., `print`, `len`)

### Example

```python
x = 10  # Global scope

def my_func():
    x = 5  # Local scope
    print(x)

my_func()
print(x)
````

---

# 2. Class

### Definition

A **class** is a blueprint for creating objects.
It defines **attributes (variables)** and **methods (functions)** that the objects created from the class will have.

### Example

```python
class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def display(self):
        print(self.name, self.age)

s1 = Student("Nitin", 22)
s1.display()
```

### Key Features

* Supports **Object-Oriented Programming (OOP)**
* Encapsulation of data and methods
* Reusable code structure

# 5. Packages

### Definition

A **package** is a collection of **multiple Python modules organized in directories**.

Packages help organize large Python projects.

Example structure:

```
project/
│
├── main.py
├── utils/
│   ├── __init__.py
│   ├── math_utils.py
│   └── string_utils.py
```

### Importing from Packages

```python
from utils.math_utils import add
```

### Benefits

* Better project organization
* Code reusability
* Easier maintenance

# 6. Packaging

### Definition

**Packaging** is the process of organizing and distributing Python code so that it can be **easily installed and reused by others**.

Python packages are often distributed using **pip**.

### Common Packaging Tools

* `setuptools`
* `pip`
* `wheel`

### Example Installation

```bash
pip install package_name
```

### Basic Package Structure

```
my_package/
│
├── setup.py
├── README.md
├── requirements.txt
└── my_module/
    ├── __init__.py
    └── module.py
```

### Benefits

* Easy distribution of code
* Dependency management
* Reusability across projects
