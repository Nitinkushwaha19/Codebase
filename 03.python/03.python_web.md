# Python Web Application

Python is widely used for web development (e.g., Instagram, Disqus).  
A typical Python web app has **three backend layers**:

---

## 1️⃣ Web Server Layer
- Handles **HTTP requests** and responses (HTML, JSON, XML).  
- Serves **static content** (images, CSS, files).  
- Popular servers: **Apache**, **Nginx**.

---

## 2️⃣ WSGI Server Layer
- **WSGI (Web Server Gateway Interface)** allows Python apps to communicate with web servers.  
- Popular WSGI servers: **Gunicorn**, **uWSGI**.  
- Benefits:
  - **Flexibility:** Decouples frameworks from web servers.  
  - **Scaling:** Handles thousands of requests, manages multiple processes.

---

## 3️⃣ Web Application Framework Layer
- Provides **tools to develop apps quickly** (templates, ORM, authentication, etc.).  
- Popular frameworks: **Django**, **Flask**, **FastAPI**.  
- Developers focus on **business logic** while framework handles common tasks.

---

# Flask

**Flask** is a **lightweight Python web framework** for building web applications quickly.

---

## 1️⃣ Hello World Example

```python id="v3k8ml"
# start.py
from flask import Flask

app = Flask(__name__)

@app.route('/')
def hello():
    return 'Hello, World!'

app.run()
````

### Explanation:

1. `Flask(__name__)` → Creates a Flask app (WSGI application).
2. `@app.route('/')` → URL `/` triggers the `hello()` function.
3. `hello()` → Returns text (HTML by default).
4. `app.run()` → Starts the server on **[http://127.0.0.1:5000/](http://127.0.0.1:5000/)**.

---

## 2️⃣ How to Run

```bash
pip install flask
python start.py
```

* Open browser: `http://127.0.0.1:5000/` → See "Hello, World!"

---

**Flask Key Points:**

* Lightweight & simple
* Ideal for small apps or APIs
* Easy to extend with libraries

# HTTP - Hypertext Transfer Protocol

HTTP is the **protocol for communication between clients (browsers) and servers** over the web.

---

## 1️⃣ HTTP Request Methods

| Method | Purpose | Notes |
|--------|---------|-------|
| `GET` | Retrieve data from a server | Does not modify server state |
| `POST` | Send data to create a resource | Non-idempotent; repeated calls create multiple resources |
| `PUT` | Send data to update/create a resource | Idempotent; repeated calls produce the same result |
| `DELETE` | Remove a resource from the server | Removes the specified resource |

---

## 2️⃣ HTTP URL

- Uniform Resource Locator (URL) specifies the **address of the resource** on the web.
- Typical structure:

```

[http://www.example.com:80/path/to/resource?query=param#fragment](http://www.example.com:80/path/to/resource?query=param#fragment)

```

Components:

| Part | Meaning |
|------|---------|
| `http://` | Protocol |
| `www.example.com` | Server domain |
| `:80` | Port (optional, default 80) |
| `/path/to/resource` | Resource path |
| `?query=param` | Query string |
| `#fragment` | Fragment identifier |

# Python Requests Library 

**Requests** is a **simple and elegant Python library** for sending HTTP requests.

---

## 1️⃣ Installation
```bash
pip install requests
````

---

## 2️⃣ Basic GET Request

```python
import requests

response = requests.get('https://api.github.com')

if response.status_code == 200:
    print('Success!')
elif response.status_code == 404:
    print('Not Found.')
```

* `response.content` → Raw bytes
* `response.text` → Response as string
* `response.json()` → Parse JSON into Python dict

---

## 3️⃣ Authentication

```python
# Basic Auth
requests.get('https://api.github.com/user', auth=('user', 'pass'))

# Token Auth
requests.get('https://git.epam.com/api/v4/projects',
             headers={'Authorization': 'Bearer myToken'})
```

---

## 4️⃣ Sending Parameters

```python
payload = {'key1': 'value1', 'key2': 'value2'}
r = requests.get('https://some.org/obj', params=payload)
print(r.url)
# Output: https://some.org/obj?key2=value2&key1=value1
```

---

## 5️⃣ Working with JSON

```python
j = [{"name": "cat", "items": [{"num": 1, "price": 30}, {"num": 2, "price": 50}]}]
price = j[0]["items"][1]["price"]  # 50
```
