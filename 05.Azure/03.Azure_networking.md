## Azure Virtual Networking

### **Overview**

- Azure Virtual Network (VNet) is the core networking service in Azure, enabling secure communication between Azure resources, users, and on-premises environments.
- VNets are customizable and scalable, supporting a wide range of networking scenarios.

---

### **Key Features**

#### **1. Isolation & Segmentation**

- VNets are isolated from each other by default.
- Custom IP address ranges (IPv4/IPv6) can be defined.
- Subnets allow logical segmentation and application of security boundaries.

#### **2. Endpoints**

- **Public Endpoints:**
  - Used for resources that need internet access.
  - Assigned public IP addresses.
- **Private Endpoints:**
  - Used for internal communication.
  - Assigned private IP addresses from VNet address space.
  - Supports Azure Private Link for secure access to Azure services.

#### **3. Name Resolution**

- Azure-provided DNS for internal name resolution.
- Custom DNS servers can be configured for advanced scenarios.

---

### **Connectivity Options**

#### **1. Internet Communications**

- Public IPs or public load balancers expose resources to the internet.
- Network security groups (NSGs) and firewalls control access.

#### **2. Communication Between Azure Resources**

- Resources within the same VNet communicate via private IPs.
- Service endpoints and Private Link enhance security and performance for Azure services.

#### **3. On-Premises Connectivity**

- **Point-to-Site VPN:**
  - Secure connection for individual clients.
- **Site-to-Site VPN:**
  - Connects entire on-premises network to Azure.
- **ExpressRoute:**
  - Dedicated, private connection for high bandwidth and security.

---

### **Routing & Traffic Management**

- **Default Routing:**
  - Azure automatically routes traffic within VNets and connected networks.
- **Custom Routing:**
  - User-defined routes (UDR) for granular control.
  - Route tables and BGP for advanced routing scenarios.

---

### **Security & Filtering**

- **Network Security Groups (NSGs):**
  - Control inbound/outbound traffic with rules based on IP, port, and protocol.
- **Network Virtual Appliances:**
  - Deploy firewalls, VPNs, and other appliances for advanced security.

---

### **Virtual Network Peering**

- Enables direct, private connectivity between VNets.
- Supports peering across regions for global network architecture.
- Traffic remains on Microsoft’s backbone network, never traversing the public internet.

---

### **Best Practices**

- Use NSGs and firewalls to protect resources.
- Segment VNets and subnets based on workload and security requirements.
- Monitor network traffic with Azure Network Watcher.
- Use Private Link and service endpoints for secure access to Azure services.
- Plan IP address ranges to avoid overlap with on-premises networks.

---

### **Useful Azure Networking Services**

- **Azure Load Balancer:** Distributes traffic across resources.
- **Azure Application Gateway:** Provides application-level routing and web application firewall.
- **Azure Firewall:** Managed, cloud-based network security service.
- **Azure Bastion:** Secure remote access to VMs without exposing them to the public internet.

# Azure Virtual Private Networks

## What is Azure VPN?

- Securely connects on-premises networks and Azure VNets over the public internet.
- Uses encrypted tunnels to protect data in transit.
- **Example:** A company links its datacenter to Azure so internal apps can access cloud resources securely.

## VPN Gateway

- Special Azure resource deployed in a dedicated subnet.
- Acts as the encrypted traffic endpoint.
- **Example:** The VPN gateway in Azure handles all encrypted traffic between your office and Azure VMs.

## Connection Types

- **Site-to-Site (S2S):** Connects on-premises datacenter to Azure VNet.
  - _Example:_ New York datacenter connects to Azure VNet for secure server communication.
- **Point-to-Site (P2S):** Connects individual devices (e.g., remote users) to Azure VNet.
  - _Example:_ Remote employee uses a VPN client to access Azure resources from home.
- **VNet-to-VNet:** Connects two Azure VNets (even across regions).
  - _Example:_ US-based VNet and Europe-based VNet communicate securely for global app deployment.

## VPN Gateway Types

- **Policy-Based:** Static IP rules for traffic encryption.
  - _Example:_ Only traffic between 10.1.0.0/16 (on-premises) and 10.2.0.0/16 (Azure) is encrypted.
- **Route-Based:** Uses routing tables; supports dynamic routing and is more flexible (preferred).
  - _Example:_ Adding a new subnet in Azure is automatically handled by the route-based VPN.

## High Availability Options

- **Active/Standby:** Two instances; standby takes over if active fails.
  - _Example:_ During maintenance, the standby VPN gateway takes over with minimal downtime.
- **Active/Active:** Both instances active; higher throughput and redundancy.
  - _Example:_ Two on-premises VPN devices each connect to a different Azure gateway instance for continuous connectivity.
- **ExpressRoute Failover:** VPN gateway as backup if ExpressRoute fails.
  - _Example:_ If the private ExpressRoute circuit goes down, traffic reroutes over the VPN gateway via the internet.
- **Zone-Redundant:** Gateways deployed across multiple availability zones.
  - _Example:_ If one Azure zone fails, the VPN gateway in another zone maintains connectivity.

## What is Azure ExpressRoute?

Azure ExpressRoute is a service that lets you connect your company’s network (like your office or datacenter) directly to Microsoft’s cloud (Azure, Microsoft 365, etc.) using a private connection. This connection is called an **ExpressRoute Circuit** and is set up with the help of a connectivity provider (like a telecom company).

**Key Point:**  
ExpressRoute connections do **not** use the public internet. Instead, they use private lines, which makes them more reliable, faster, and secure.

---

### Features and Benefits

- **Direct Access:** Connects you to Microsoft services like Azure, Office 365, and Dynamics 365.
- **Global Connectivity:** With ExpressRoute Global Reach, you can connect multiple offices or datacenters around the world through Microsoft’s network.
- **Dynamic Routing:** Uses BGP (Border Gateway Protocol) for automatic route updates between your network and Microsoft’s cloud.
- **Built-in Redundancy:** Providers use backup devices and you can set up multiple circuits for high reliability.

---

### When Should You Use ExpressRoute?

- You need a private, reliable connection between your company and Azure.
- You have strict compliance or security requirements.
- You want predictable, high-speed network performance.
- You want to avoid sending sensitive data over the public internet.

# Azure DNS – Quick Notes

## What is Azure DNS?

- A Microsoft service for hosting and managing DNS domains and records.
- Provides name resolution for resources inside and outside Azure.

## Key Benefits

- **Reliability & Performance:** Uses Azure’s global DNS servers and anycast for fast, highly available responses.
  - _Example:_ Users in Europe get fast DNS resolution for your Azure-hosted website.
- **Security:** Integrated with Azure RBAC, activity logs, and resource locks.
  - _Example:_ Only IT admins can update DNS records; all changes are logged.
- **Ease of Use:** Manage DNS via Azure portal, CLI, PowerShell, REST API, or SDKs.
  - _Example:_ Update DNS records for your website using the Azure portal or scripts.
- **Custom Private Domains:** Supports private DNS zones for internal Azure networks.
  - _Example:_ Use `app1.internal.company.com` for VMs in your virtual network.
- **Alias Records:** DNS records that point to Azure resources and auto-update if IPs change.
  - _Example:_ `www.company.com` alias record points to an Azure public IP or Traffic Manager.

## Common Services Used With Azure DNS

- Azure Web Apps/App Services
- Azure Virtual Machines
- Azure Traffic Manager
- Azure CDN
- Azure Private Link/Private Endpoints
