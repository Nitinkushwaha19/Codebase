# Git Definition
```bash
Git is a version control system that helps track changes in code and lets multiple people work on the same project safely and efficiently.
```


# ⚙️ Setup & Initialization

## 🔹 `git config`

Configures Git settings like username, email, and editor for your system or repository.

```bash
git config --global user.name "Nitin Kushwaha"
```

---

## 🔹 `git init`

Creates a new Git repository in the current directory.

```bash
git init
```

---

## 🔹 `git clone`

Copies an existing remote repository to your local machine.

```bash
git clone https://github.com/user/repo.git
```

---

# 📂 Staging & Committing

## 🔹 `git status`

Displays the current state of tracked and untracked files.

```bash
git status
```

---

## 🔹 `git add`

Stages changes so they can be included in the next commit.

```bash
git add .
```

---

## 🔹 `git commit`

Saves staged changes as a new commit in the repository history.

```bash
git commit -m "Add authentication feature"
```

---

## 🔹 `git diff`

Shows differences between working directory, staging area, or commits.

```bash
git diff
```

---

## 🔹 `git log`

Displays commit history with details.

```bash
git log --oneline
```

---

# 🌍 Remote Repositories

## 🔹 `git remote`

Manages remote repository connections like adding or viewing remotes.

```bash
git remote -v
```

---

## 🔹 `git push`

Uploads local commits to a remote repository branch.

```bash
git push origin main
```

---

## 🔹 `git pull`

Fetches and merges changes from a remote branch into your current branch.

```bash
git pull origin main
```

---

## 🔹 `git fetch`

Downloads changes from a remote repository without merging them.

```bash
git fetch origin
```

---

# 🌿 Branching & Merging

## 🔹 `git checkout`

Switches branches or restores files to a previous state.

```bash
git checkout develop
```

Create and switch branch:

```bash
git checkout -b feature/login
```

---

## 🔹 `git merge`

Combines changes from one branch into another branch.

```bash
git merge feature/login
```

---

# 🔄 Rebase & History Management

## 🔹 `git rebase`

Moves your commits on top of another branch to create a clean, linear history.

```bash
git rebase main
```

---

## 🔹 `git rebase -i`

Interactively edits, reorders, squashes, or removes past commits.

```bash
git rebase -i HEAD~3
```

---

# 🎯 Cherry-Pick & Tags

## 🔹 `git cherry-pick`

Applies a specific commit from another branch onto your current branch.

```bash
git cherry-pick a1b2c3f
```

---

## 🔹 `git tag`

Creates a fixed reference to a specific commit, usually for releases.

```bash
git tag -a v1.0 -m "Release v1.0"
```

Push tags:

```bash
git push origin --tags
```

---

# 📦 Stashing

## 🔹 `git stash`

Temporarily saves uncommitted changes and cleans the working directory.

```bash
git stash push -m "WIP payment module"
```

Apply stash:

```bash
git stash pop
```

---

# 🧹 Cleanup & Undo

## 🔹 `git reset`

Moves HEAD to another commit and optionally unstages or discards changes.

```bash
git reset --hard HEAD~1
```

---

## 🔹 `git revert`

Creates a new commit that safely undoes a previous commit.

```bash
git revert a1b2c3f
```

---

## 🔹 `git clean -xdf`

Deletes all untracked files and directories, including ignored files.

```bash
git clean -xdf
```
