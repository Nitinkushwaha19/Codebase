## Control Plane (Master Node Components)
The Master Node manages the Kubernetes cluster. It runs the control plane components (API Server, etcd, Scheduler, Controller Manager, Cloud Controller Manager) that make decisions about the cluster and maintain its desired state.

### API Server

The **API Server** is the central management component that exposes the Kubernetes API. It validates and processes requests from users, CLI tools, and other components, updating the cluster state in **etcd**.

### etcd

**etcd** is a **distributed key-value store** that stores all cluster data, including configuration, state, metadata, and information about nodes, pods, and services.

### Scheduler

The **Scheduler** assigns newly created Pods to suitable worker nodes based on resource availability, policies, constraints, and scheduling rules.

### Controller Manager

The **Controller Manager** runs a set of controllers that continuously monitor the cluster to **ensure the actual state matches the desired state**.

### Cloud Controller Manager

The **Cloud Controller Manager** integrates Kubernetes with the underlying cloud provider (AWS, Azure, GCP) to manage cloud-specific resources like load balancers, storage volumes, and nodes.

## Worker Node Components
A Worker Node runs application workloads (Pods). It contains kubelet, kube-proxy, and a container runtime to execute containers and handle networking, ensuring Pods are healthy and reachable.

### kubelet

The **kubelet** is an agent running on each worker node that ensures containers described in Pod specifications are running and healthy.

### kube-proxy

**kube-proxy** maintains network rules on each node to enable **communication between Pods and services**, handling service discovery and load balancing.

### Container Runtime

A **Container Runtime** is the software responsible for **running containers** on a node, managing their lifecycle, and pulling container images. Examples: Docker, containerd, CRI-O.

## Other Components

### Pod

A **Pod** is the **smallest deployable unit in Kubernetes**, consisting of one or more containers that share network and storage resources.

### Service

A **Service** exposes a set of Pods as a **network service**, providing a stable IP and DNS name for access.

### Volume

A **Volume** is a **persistent storage resource** that can be mounted into a Pod to store and share data beyond the container lifecycle.

### Namespace

A **Namespace** is a logical partition within a cluster that **organizes and isolates resources among multiple users, teams, or projects**.

### Ingress

An **Ingress** manages external HTTP and HTTPS access to services, providing **routing, load balancing, and SSL termination**.
