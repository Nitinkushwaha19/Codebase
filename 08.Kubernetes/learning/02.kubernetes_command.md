# Kubernetes Commands Cheat Sheet

## 1. **kubectl get**

**Definition:** Fetches resources from the cluster.

**Syntax:**

```bash
kubectl get <resource> [name] [flags]
```

**Attributes / Flags:**

* `-o` → output format (`yaml`, `json`, `wide`)
* `--all-namespaces` → list resources from all namespaces

**Example:**

```bash
kubectl get pods
kubectl get nodes -o wide
kubectl get services --all-namespaces
```

---

## 2. **kubectl describe**

**Definition:** Shows detailed information about a specific resource.

**Syntax:**

```bash
kubectl describe <resource> [name] [flags]
```

**Example:**

```bash
kubectl describe pod nginx-pod
kubectl describe node node1
```

---

## 3. **kubectl create**

**Definition:** Creates a resource from a file or command.

**Syntax:**

```bash
kubectl create -f <filename>
kubectl create <resource> <name> [flags]
```

**Example:**

```bash
kubectl create -f pod.yaml
kubectl create namespace dev
kubectl create deployment nginx-deploy --image=nginx
```

---

## 4. **kubectl apply**

**Definition:** Applies configuration to a resource. Updates if it already exists.

**Syntax:**

```bash
kubectl apply -f <filename>
```

**Example:**

```bash
kubectl apply -f deployment.yaml
```

---

## 5. **kubectl delete**

**Definition:** Deletes resources from the cluster.

**Syntax:**

```bash
kubectl delete <resource> [name] [flags]
kubectl delete -f <filename>
```

**Example:**

```bash
kubectl delete pod nginx-pod
kubectl delete -f service.yaml
kubectl delete namespace dev
```

---

## 6. **kubectl logs**

**Definition:** Fetches logs from a container in a Pod.

**Syntax:**

```bash
kubectl logs <pod-name> [-c container-name] [flags]
```

**Example:**

```bash
kubectl logs nginx-pod
kubectl logs nginx-pod -c nginx-container
```

---

## 7. **kubectl exec**

**Definition:** Executes a command in a running container inside a Pod.

**Syntax:**

```bash
kubectl exec -it <pod-name> -- <command>
```

**Example:**

```bash
kubectl exec -it nginx-pod -- /bin/bash
kubectl exec -it mypod -- ls /usr/share/nginx/html
```

---

## 8. **kubectl scale**

**Definition:** Scales a deployment, replicaset, or replication controller.

**Syntax:**

```bash
kubectl scale <resource>/<name> --replicas=<number>
```

**Example:**

```bash
kubectl scale deployment/nginx-deploy --replicas=5
```

---

## 9. **kubectl expose**

**Definition:** Exposes a resource as a Kubernetes Service.

**Syntax:**

```bash
kubectl expose <resource>/<name> --port=<port> --target-port=<target-port> [--type=<type>]
```

**Example:**

```bash
kubectl expose deployment/nginx-deploy --port=80 --target-port=80 --type=LoadBalancer
```

---

## 10. **kubectl port-forward**

**Definition:** Forwards a local port to a port on a Pod or Service.

**Syntax:**

```bash
kubectl port-forward <pod/service> <local-port>:<remote-port>
```

**Example:**

```bash
kubectl port-forward pod/nginx-pod 8080:80
kubectl port-forward service/nginx-service 9090:80
```

---

## 11. **kubectl apply edit / patch**

**Definition:** Updates a resource using interactive editor or patch.

**Syntax:**

```bash
kubectl edit <resource>/<name>
kubectl patch <resource>/<name> --patch '{"spec":{"replicas":3}}'
```

**Example:**

```bash
kubectl edit deployment/nginx-deploy
kubectl patch deployment/nginx-deploy --patch '{"spec":{"replicas":4}}'
```

---

## 12. **kubectl rollout**

**Definition:** Manages deployment rollout operations.

**Syntax:**

```bash
kubectl rollout status deployment/<name>
kubectl rollout undo deployment/<name>
kubectl rollout history deployment/<name>
```

**Example:**

```bash
kubectl rollout status deployment/nginx-deploy
kubectl rollout undo deployment/nginx-deploy
kubectl rollout history deployment/nginx-deploy
```

---

## 13. **kubectl top**

**Definition:** Shows metrics like CPU and memory usage for nodes or pods (requires metrics-server).

**Syntax:**

```bash
kubectl top pod [pod-name]
kubectl top node [node-name]
```

**Example:**

```bash
kubectl top pod nginx-pod
kubectl top node node1
```

---

## 14. **kubectl get all**

**Definition:** Lists **all resource types** in a namespace.

**Syntax:**

```bash
kubectl get all [flags]
```

**Example:**

```bash
kubectl get all
kubectl get all --all-namespaces
```

---

## 15. **kubectl explain**

**Definition:** Displays documentation of a Kubernetes resource or field.

**Syntax:**

```bash
kubectl explain <resource> [field]
```

**Example:**

```bash
kubectl explain pod
kubectl explain pod.spec.containers
```

---

## 16. **kubectl config**

**Definition:** Manages kubeconfig files and contexts for multiple clusters.

**Syntax:**

```bash
kubectl config view
kubectl config use-context <context-name>
kubectl config set-context <context-name> --namespace=<namespace>
```

**Example:**

```bash
kubectl config view
kubectl config use-context minikube
kubectl config set-context dev-context --namespace=dev
```

---

## 17. **kubectl cluster-info**

**Definition:** Displays cluster endpoint info.

**Syntax:**

```bash
kubectl cluster-info
```

**Example:**

```bash
kubectl cluster-info
```
